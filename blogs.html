  1. 1. Differences between map, filter, forEach & find :

    - For manipulating arrays, JavaScript provides the map, filter, forEach and find methods. Here are the differences between them :

    1. map() : The map method applies a function to each element of the initial array to produce a new array. The function receives the current value of the element, its index, and the array itself as parameters. A new value is returned by the function, and it is added to the new array.
    2. filter() : The original array's elements that don't match a specific condition are excluded using the filter method. The function receives the current value of the element, its index, and the array itself as parameters.
    3. forEach() : The forEach method applies a function to each array element. The element's current value, index, and the array itself are passed as parameters to the function. The function doesn't return anything.
    4. find() : The first element of the array that matches a specific condition is returned by the find method. The function receives the current value of the element, its index, and the array itself as parameters.
  2. 2. Differences between local storage, session storage and cookies :

    - Local storage, session storage and cookies are all methods of storing data in a web browser, but each has unique characteristics and applications. The differences between them are as follows: :

    1. LocalStorage : LocalStorage is a data storing mechanism which stores data on the client's web browser until the user or an application clears it. Even if the user quits the browser or restarts their device, the data is still accessible. Since LocalStorage uses a key-value pair storage system, it stores data as strings.
    2. SessionStorage : SessionStorage is similar to LocalStorage but after the user closes the browser or the tab, or turns off the device, the data is deleted. Along with a key-value pair storing method, SessionStorage keeps data as strings.
    3. cookies : Cookies are small files that are sent to the server with each request and saved on the client's web browser. Cookies can be used to save user preferences or login information and have an expiration date. Cookies can only hold a maximum of 4KB of data.
  3. 3. Event Loop :

    - The way that JavaScript manages and runs code is known as an event loop in programming. JavaScript code can only do one thing at a time since it runs in a single-thread. The event loop manages and prioritizes tasks in order to make the code execution as efficient as possible.

    The task queue and call stack are continuously monitored by the event loop. Call stack is the data structure that maintains track of the ongoing function calls. Callback functions and other tasks that must be executed are stored in the task queue.

    whenever a function is called, it is added to the call stack. If the function contains asynchronous code, such as a setTimeout, it is moved to the task queue rather than blocking the call stack. The event loop checks the task queue and adds the next task to the call stack when the call stack is empty. This process is repeated until all tasks are finished.
